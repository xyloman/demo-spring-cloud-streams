## Example Spring Cloud Streams Application

This application is a simple spring cloud streams application that contains a rest endpoint `/queue-message` that consumes a string request payload that is placed onto a queue using the EventSource class.  The message is then consumed from the Sink and logged out using slf4j.  

The purpose of this demo application is to show that if you continuously hist the `/queue-message` with load such as with the following script:

```
for ((i=1;i<=10000;i++)); do curl https://demo-spring-cloud-streams-reflective-bandicoot.apps.dev.us-west-2.cernercf.io/queue-message -H 'Content-Type: text/plain' -d "hello:${i}"; sleep(5); done
```

That the REST request would fail to produce the message when the RabbitMQ cluster is restarted.  This application has been tested on the following software:

- Pivotal Cloud Foundry 2.1.9 
- Uses the service binding framework from CloudFoundry see manifest.yml
- Bound to a On Demand RabbitMQ 3.7 cluster with 3 nodes 
- RabbitMQ tile version is 1.12.7

To test turbulence in the deployment we are leveraging the following two bosh commands to simulate a stemcell or RMQ upgrade when we first noticed the issue:

```
bosh -d service-instance_<service-instance-guid> restart
bosh -d service-instance_<service-instance-guid> recreate
```
